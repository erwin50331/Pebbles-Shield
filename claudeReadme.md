專案名稱：鵝卵石之盾 (Pebbles Shield) - 自動化次文化趨勢監控與 Discord 防護系統

專案背景：
我目前擔任一個 VTuber 社群（「黑天鵝」）的頻道管理員（角色：FBI）。由於網路次文化惡意隱語（如：淫夢梗、歧視性諧音）演變極快，傳統的靜態黑名單已不足夠。我需要開發一套能自動追蹤網路趨勢並偵測 Discord 惡意言論的系統。

開發目標：
建立一個包含「自動化爬蟲」、「語義分析」與「管理儀表板」的整合系統。系統需能主動從社群平台抓取新興黑話，並即時應用於 Discord 訊息監控，同時提供「心理防禦」介面。

核心功能需求：

1. 多平台趨勢爬蟲 (Trend Scraper)：

    目標平台： 定時爬取 PTT（如 Gossiping, C_Chat 板）、巴哈姆特（場外休憩區）、Threads 等平台的熱門討論。

    偵測邏輯： 抓取出現頻率突然增高的詞彙（Trending Words），特別是與性少數、歧視、或特定惡質迷因語法相關的詞彙。

    自動更新： 將偵測到的疑似惡意新詞彙存入「待審核資料庫」，而非僅依賴手動輸入。

2. 智能語義辨識引擎：

    初步篩選： 結合靜態已知黑名單（如：池沼、Homo 特有、兩女一腸、(確信)）作為基準。

    AI 延伸分析： 串接 LLM API（如 Claude API），對爬蟲抓到的新詞進行語義分析，判斷其是否帶有惡意、歧視或特定的負面次文化背景，並給予風險評分。

3. Discord 即時監控與過濾：

    即時監聽： 透過 Discord API 監看指定頻道。

    動態比對： 比對訊息是否包含「已知黑名單」或「高風險趨勢詞」。

    風險預警： 在儀表板即時標記高風險發言。

4. 管理儀表板 (Dashboard) 與心理防禦：

    趨勢看板： 顯示目前網路上新興的疑似惡意黑話。

    心理防禦模式： 提供「遮蔽原文」功能。開啟後，管理員在儀表板看到的違規訊息會被自動替換為「語義類型」（例如：[偵測到新興歧視語 - 來源：PTT]），避免直接接觸有害內容。

    一鍵管理： 整合刪除訊息、剔除 (Kick)、封禁 (Ban) 功能。

給 Claude Code 的執行步驟：

    第一步： 建立基礎的 Discord Bot 監控架構。

    第二步： 開發針對 PTT 或巴哈姆特的基礎爬蟲模組（使用 axios 或 puppeteer），實作關鍵字頻率統計。

    第三步： 實作語義分析邏輯，將爬蟲結果與 Discord 監控連結。

    第四步： 開發 React 儀表板，包含「趨勢管理」與「心理防禦過濾」功能。
